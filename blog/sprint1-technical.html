Technical Blog #1: Positioning of Elements

04 September 2020

To see how things are styled (made to look just how they do) on a webpage it can be helpful to consider that everything on a web page consists of boxes. Each element on a web page has it’s own rectangle (or sometimes square) that surrounds it. Any elements that are inside another element are referred to as a child element, while elements that encompass other elements are referred to as parent elements. We cannot necessarily see these boxes but if we were to add a border to each element these rectangles would become visible. Instead we can visualise these boxes in our head (or physically draw them) to help understand the positioning of elements. The position property is used to position elements using CSS (Cascading Style Script) code however the position property has six different values that can be used. As a rule the default position value for new elements is static (which just means the elements will flow into the page as normal).


Relative, Fixed and Absolute Positioning

What do they do and how do they differ?

Relative Positioning:

I want to position an element relative to where it would normally be


Simply setting the position: relative value to an element will do nothing. You must supply a positioning element (top, left, bottom or right). The element will then position relative to where it would normally be. e.g. top:10px will position the element 10px (pixels) down from where it would normally be. Think of it as adding a 10 pixel gap in place of where the element would be, pushing the element down.
Position Relative also introduces the ability to use z-index (stack order) on that element. The easy way to visualise stack order is to think of sheets of paper. They may not all be the same size or shape, but when you stack them on top of each other some parts may be covered by other pieces of paper, while some pieces of paper might be completely covered and not visible at all. The top of the pile (the element with the highest z-index) will always be completely visible. It’s worth knowing that z-index only works on positioned elements (absolute, relative, fixed, sticky), so this element will now appear on top of any static elements.
